Exit Test: Order Processing System Enhancement

Duration  : 60 Minutes

Scenario

You are part of an e-commerce development team. Customers have reported that the 
final order amount is sometimes incorrect. The application calculates discounts and shipping
 charges based on order value and customer type.

Your task is to analyze the existing code, identify issues, implement new business requirements,
 and validate your solution using GitHub Copilot. 

 
Files Provided

OrderService.java

 import java.util.List;

public class OrderService {

    public double calculateTotal(List<Double> items, String customerType) {

        double total = 0;

        for (Double item : items) {
            total += item;
        }

        double discount = 0;

        if (customerType.equals("PREMIUM")) {
            discount = total * 0.15;
        }

        if (total > 1000) {
            discount = total * 0.10;
        }

        double shippingCharge = 50;

        if (total > 500) {
            shippingCharge = 0;
        }

        return total - discount + shippingCharge;
    }
}
 
 
Business Requirements

Existing Rules

* Orders above ₹500 receive free shipping.
* PREMIUM customers receive 15% discount.

 New Requirements

1. PREMIUM customers get 15% discount.
2. GOLD customers get 10% discount.
3. SILVER customers get 5% discount.
4. Orders above ₹1000 get an additional 10% discount.
5. Total discount cannot exceed 20%.
6. Invalid customer types should be handled gracefully.
7. Empty or null order lists should not break the application.
